# Example Output

## Project Summary

The client manages inbound maintenance requests through email and phone calls, then tracks status manually in a spreadsheet. The main need is a cleaner intake process, a visible work queue, and consistent status updates.

## Primary Business Goal

Reduce missed requests and give staff a simple way to track maintenance work from intake to completion.

## Users / Roles

- tenant
- office staff
- maintenance coordinator
- manager

## Core Entities

- property
- unit
- tenant
- maintenance request
- status update
- assigned technician

## Required Forms

- maintenance request form
- internal assignment form
- completion/update form

## Required Views / Dashboards

- open requests queue
- overdue requests view
- requests by property
- weekly summary dashboard

## Workflow Stages

- submitted
- reviewed
- assigned
- scheduled
- completed
- closed

## Notifications / Alerts

- confirmation to tenant
- assignment notice to staff
- overdue reminder

## Reports

- open vs completed requests
- average completion time
- requests by property

## Recommended Implementation Approach

Start with a Google Form plus an internal tracking dashboard if budget is tight. Move to a portal/dashboard app if the team needs permissions, richer statuses, and better reporting.

## Estimated Complexity

small
