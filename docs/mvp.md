# MVP

## Goal

Get to paid workflow consulting as quickly as possible.

The MVP is not a full builder platform. It is a workflow discovery and specification pipeline.

## Customer Problem

Small businesses already feel pain around:

- manual spreadsheet updates
- email handoffs
- status ambiguity
- duplicated entry across tools
- missed follow-ups

They are not looking for AI for its own sake. They are looking for operational relief.

## MVP Promise

Turn a messy business process into:

- a clear workflow summary
- a list of entities and roles
- a suggested internal tool design
- a rough implementation plan
- a quoteable scope

## MVP Deliverables

For each prospect:

1. Intake response
2. AI-generated workflow summary
3. AI-generated spec
4. Human-reviewed proposal
5. Build recommendation

## What To Do Manually At First

Keep these manual early:

- qualifying leads
- reviewing generated summaries
- adjusting scope
- estimating pricing
- deciding whether to build in Sheets, a portal, or a custom app

Manual work is a feature in Phase 0 because it helps sharpen the interview engine.

## Success Criteria

The MVP is working when:

- a prospect can describe their current process in plain language
- the system converts that into a coherent workflow artifact
- you can turn that artifact into a paid proposal in under 1 hour
- at least one customer agrees to pay for implementation
